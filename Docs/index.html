Get Started

All you need to do before starting work with Smooth PHP Framework is to carefull read the documentation of our product and in case of any issues to contact our team.

Smooth Team wishes you all the best using our product.

Overview

Smooth is a new and sparkling PHP framework who is trying to ease and quicken your work process, because we now that writing faster is really important to all developers.

Change Log

This is our first version so everything is sparkling new.

Requirements

Smooth PHP is requiring only Apache and SQL server(if you want to use databases) plus PHP version greater or equal to 5.3.0.

Installation

Everything is that simple. You just have to put all the Smooth files to your server, usually index.php is in your root directory; In order to configure your database connection you have to run the file named Database.php in Smooth/database folder and fill your server connection.

Configuration

Variables

BASEPATH

Path yo your project

APPPATH

Path to your app folder with the models, views, controllers

SYSPATH

Path you your Smooth folder with all files need for proper working process

WEBPATH

Path to your project optimized for web usage

CONTROLLER

The name of your base controller

SMOOTH_V

The version of Smooth you are using

Model View Controller

Model

The models are PHP files, designed to work with database information. Here is a sample model:

  1. <?php
  2.  use Smooth\Libraries\Db;
  3.  class SampleModel extends Model {
  4.   public function fetch_users() {
  5.    return Db::fetch('users');
  6.   }
  7.  }
  8. ?>

View

The views are PHP files which are responsible for presenting the final information to the users. These pages can be different parts(header,section,nav,footer,sidebar) or entire web pages. The choice is completely yours. The views can contain any information you want to.

Controller

The controllers are PHP files linked with defined url. So if you want to get to the home view of your site you should http://mysite.com/home which is equal to http://mysite.com/index.php/home. Here is one sample controller:

  1. <?php
  2.  class SampleController extends Controller {
  3.   public function index() {
  4.    $data['content'] = 'WelcomeSmoothView';
  5.    $this->render('includes/template', compact('data'));
  6.   }
  7.  }
  8. ?>

If your will is to create a new cookie use our set function by passing cookie name and value parameters. The third one is the validity period, it is optional and is set to 3600 by default.

  1. Cookie::set('my_cookie', 'Smooth');
get(name)
  1. Cookie::get('my_cookie');
delete(name)
  1. Cookie::delete('my_cookie');

Db

Use of Db class to smoothen your work process with databases. Smooth is using PDO, however you will not be pushed to write SQL queries if you don't want to. Load it with:

  1. use Smooth\Libraries\Db;
query(SQL query)

Want to write all your queries by your own. The decision is yours.

  1. Db::query("SELECT user,name, email from users");
fetch(table, limit=null)

Function selecting all the data from your aimed table. The second parameter is limit and is set to null by default, but if you want to limit the result just put a number after a comma.

  1. Db::fetch('users');
  2. //equal to
  3. Db::query("SELECT * FROM users");
fetch_where(table, array params, limit = null)

Function which will help you in select data with where statement, again our limit parameter is optional.

  1. $params = array('user'=>'me', 'email'=>'myemail@mysite.com');
  2. Db::fetch_where('users', $params);
  3. //equal to
  4. Db::query("SELECT * FROM users WHERE user = 'me' AND email = 'myemail@mysite.com'");
fetch_or_where(table, array params, limit = null

Function similar to fetch_where but added OR instead of AND. Our third parameter(limit) is set to null by default and is optional.

  1. $params = array('user'=>'me', 'email'=>'myemail@mysite.com');
  2. Db::fetch_or_where('users', $params);
  3. //equal to
  4. Db::query("SELECT * FROM users WHERE user = 'me' OR email = 'myemail@mysite.com'");
fetch_where_in(table, field, array params, limit = null)

Function making you capable of specifying multiple values in WHERE clause. Sounds a bit perplexing? Look at your example code:

  1. $params = array('me', 'you', 'them');
  2. Db::fetch_where_in('users', 'user', $params);
  3. //equal to
  4. Db::query("SELECT * FROM users WHERE user IN ('me', 'you', 'them')");

In this case you need to pass your table name, field name and the multiple values in a array, again our limit is set to null by default and is optional, however it is the 4 parameter in this function.

fetch_where_not_in(table, field, array params)

This function syntax is similar to fetch_where_in except it adds NOT and changes the direction of our query.

  1. $params = array('me', 'you', 'them');
  2. Db::fetch_where_not_in('users', 'user', $params);
  3. //equal to
  4. Db::query("SELECT * FROM users WHERE user NOT IN ('me', 'you', 'them')");
fetch_max(table, field, as = null)

Function that returns the largest value of the column.

  1. Db::fetch_max('users', 'user');
  2. //equal to
  3. Db::query("SELECT MAX('user') FROM users");

The third parameter is the as part and is set to null by default which is making it optional. Use it in case you need giving a field another name in your queries.

fetch_min(table, field, as = null)

The function is similar to fetch_max, however it returns the smallest values of the column.

  1. Db::fetch_min('users', 'user');
  2. //equal to
  3. Db::query("SELECT MIN('user') FROM users");

The third parameter is the as part and is set to null by default which is making it optional. Use it in case you need giving a field another name in your queries.

fetch_average(table, field, as = null)

The function syntax is similar to the fetch_max and fetch_min, however it returns the average value of a numeric column.

  1. Db::fetch_average('users', 'user');
  2. //equal to
  3. Db::query("SELECT AVG('user') FROM users");

The third parameter is the as part and is set to null by default which is making it optional. Use it in case you need giving a field another name in your queries.

fetch_sum(table, field, as = null)

The function syntax is similar to the other functions, however it returns the total sum of a numeric column.

  1. Db::fetch_sum('users', 'user');
  2. //equal to
  3. Db::query("SELECT SUM('user') FROM users");

The third parameter is the as part and is set to null by default which is making it optional. Use it in case you need giving a field another name in your queries.

fetch_count(table, field, as = null)

The function syntax is similar to the other functions, however it returns the number of values of the column.

  1. Db::fetch_sum('users', 'user');
  2. //equal to
  3. Db::query("SELECT SUM('user') FROM users");

The third parameter is the as part and is set to null by default which is making it optional. Use it in case you need giving a field another name in your queries.

Note: The functions does not count NULL values.

insert(table, array params)

The function is helpful when insert new rows into the table without needing a dozen lines of code.

  1. $params = array('user'=>'me', 'email'=>'mymail@mysite.com', 'name'=>'Joe', 'surname'=>'Anderson');
  2. Db::insert('users', $params);
  3. //equal to
  4. Db::query("INSERT INTO users (user, email, name, surname) VALUES ('me', 'mymail@mysite.com', 'Joe', 'Anderson')");

Note: The column names must match those given in the array.

update(table, array params, array matches)

The function is helpful when updating existing rows.

  1. $params = array('user'=>'me', 'email'=>'mymail@mysite.com', 'name'=>'John', 'surname'=>'Smith');
  2. Db::update('users', $params, array('user'=>'Joe','surname'='Anderson'));
  3. //equal to
  4. Db::query("INSERT INTO users (user, email, name, surname) VALUES ('me', 'mymail@mysite.com', 'Joe', 'Anderson') WHERE user='Joe' AND 'surname='Anderson'");

Note: The column names must match those given in the array.

delete(table, array params)

Function responsible for deleting rows from your table. Just pass your table name and your where parameters.

  1. $params = array('user'=>'me', 'surname'=>'Anderson');
  2. Db::delete('users', $params);
  3. //equal to
  4. Db::query("DELETE FROM users WHERE user='me' AND surname='Anderson");

Note: The column names must match those given in the array.

truncate(table)

With this function you are able to delete all of the rows in a table and return the counting to 0.

  1. Db::truncate('users');
  2. //equal to
  3. Db::query("TRUNCATE users");

Form

Our Form library is containing some functions helpful when generating and submiting forms withou writing not a single line of code. Loading the Form library is getting done with this line of code:

  1. use Smooth\Libraries\Form;
filter(field name, rules)

The filter function is used when filtering data values from input fields.

  1. Form::filter('username', rules);

The rules are separated with . between them. Currently those below are included in the package.

alpha
email
numeric
ip
alpha_numeric
alpha_dash_under
  1. Form::filter('username', alpha.email.ip);

Note: Form input field names must match those into the filter function.

length(field, min_length, max_length)

Length function is used for setting up minimum and maximum length of the input field.

  1. Form::filter('username', 5, 30);

Note: Form input field names must match those into the filter function.

check()

Check function is used when validating the submition of the user form, by checking all custom rules set by the developer.

  1. Form::filter('username', alpha_numeric);
  2. Form::filter('email', email);
  3. Form::length('username', 5, 30);
  4. Form::length('email', 10, 50);
  5. if( Form::check() == true ) {
  6.  //rest of your code
  7. }

If any errors apper the statement of the check function will return false.

post(field)

Post function is used when retaining the value of a field in a form with post method.

  1. Form::post('username');

Note: Form input field names must match those into the filter function.

get(field)

Get function is used when retaining the value of a field in a form with get method.

  1. Form::get('username');

Note: Form input field names must match those into the filter function.

open(action, method, id, class)

The function is used when generating forms without writing a html code.

  1. Form::open('url to the requested action file', 'post|get', 'form-wanted-id', 'form-wanted-class');

Routing

Smooth Routing system is pretty simple. For example if you want to load the home controller, add function and pass an id in order to run your func() propertly you need to run the above in your browser.

  1. http://smoothphp.com/home/add/5

Template System

Our custom template system integrated into the Smooth version is pretty simple and quite flexible. The main files can be found onto this path Application/views/includes/. There are three files header.php, footer.php and template.php. How to use the template system? Here is an example:

  1. <?php
  2.  class SampleController extends Controller {
  3.   public function index() {
  4.    $data['content'] = 'WelcomeSmoothView';
  5.    $this->render('includes/template', compact('data'));
  6.   }
  7.  }
  8. ?>

In this way you will load the view called WelcomeSmoothView and pass all the vars associated with the $data. In case you want to pass more vars containing some information you need to use this way:

  1. $data['my_var'] = 'my-information';
  2. $data['content'] = 'WelcomeSmoothView';
  3. $this->render('includes/template', compact('data'));

In your view file you can retain the data form the variable by just write its name with a $ sign in front.

  1. echo $my_var; //Will output my-information

App

Our app folder is called Application and it contains all the user files. Mostly is used to handle the MVC files(Models, Views, Controllers).
